Clarifying in-app messaging governance

Eliminating redundant comms by fixing the intake process

The problem

POs submitted requests to marketing and UX separately. Each assessed using different criteria. UX had no authority to decline. Result: redundant messages across channels, client confusion, frequent escalations.

New workflow

PO
UX
Comms
Escalation

Criteria for in-app messaging

All must be true:

  1. At least 5k clients or 10k users affected
  2. We know the exact screen where they need this info
  3. User will stop their current task to address it

Opportunity: Create a one-page criteria checklist or decision tree that POs can reference during intake.

UX decision: value exceeds disruption

If criteria 1-3 pass, UX assesses: does the value delivered (time or money saved × impacted users) outweigh the disruption cost (interruption cost × total viewers)?

Opportunity: Create a simple calculator or 2x2 matrix showing high-value/low-disruption (approve), low-value/high-disruption (decline), and edge cases.

Cross-functional alignment

  • Co-created framework with comms and product ops
  • Intake requires one checkbox: "candidate for in-app?"
  • PO facilitates one discovery, not two
  • One unified answer replaces parallel assessments

Impact

  • Eliminated redundant messaging across channels
  • Addressed client pain: "You tell me so many things in so many places"
  • Removed UX's gatekeeping role without authority
  • Dropped exec escalations
  • Improved UX efficiency by removing unnecessary cycles
  • Strengthened governance without process overhead
