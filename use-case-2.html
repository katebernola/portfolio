Clarifying in-app messaging governance

UX leadership — establishing cross-functional governance

The problem

Product owners submitted messaging requests to marketing and UX separately. Each group assessed using different criteria. UX had no authority to decline. Result: redundant messages across channels, client confusion ("You tell me so many things in so many places"), and frequent escalations.

New workflow

Single intake through marketing, shared decision criteria, UX consulted only when appropriate.

In-app messaging criteria

All three must be true:

  1. At least 5,000 clients or 10,000 users affected
  2. We know the exact screen where users need this information
  3. User will stop their current task to address it immediately

If criteria 1 or 2 is no, the message is not a candidate for in-app. UX is not engaged.

UX decision: value exceeds disruption

For messages that meet criteria 1–3, UX assesses: does the value delivered (time or money saved × impacted users) outweigh the disruption cost (interruption cost × total viewers)?

This gate prevents well-intentioned messages from interrupting users when the benefit doesn't justify the interruption.

Cross-functional alignment

  • Co-created framework with marketing and product ops
  • Intake requires one checkbox: "Candidate for in-app?"
  • Product owner facilitates one discovery conversation instead of two
  • One unified answer replaces parallel assessments
  • Marketing owns channel strategy; UX owns placement and user impact

Outcomes

  • Eliminated redundant messaging across channels
  • Addressed the client pain point directly
  • Removed UX's gatekeeping role without authority
  • Reduced exec escalations to match marketing's historical baseline
  • Improved UX efficiency by removing unnecessary intake cycles
  • Strengthened governance without adding process overhead
  • Increased consistency and clarity across all client communications
